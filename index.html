# What is Git?

        
Git is a distributed version control system that tracks changes, enables branching, and supports collaboration.

# Git Best Practices

        
Do’s and Don’ts: commit messages, branch naming, .gitignore, README habits.

      


      

        

          

            

              
Do: Clear commit messages

              
Use imperative mood + explain intent (what + why).

              
hover to flip →

            

            

              
Example

              
feat(auth): add login validation
- prevent empty submissions
- show inline error

            

          

        


        

          

            

              
Do: Good branch naming

              
Use prefixes like feature/, fix/, chore/.

              
hover to flip →

            

            

              
Example

              
feature/profile-page
fix/navbar-spacing
chore/update-deps

            

          

        


        

          

            

              
Don’t: Commit secrets

              
Add .env, keys, build outputs to .gitignore.

              
hover to flip →

            

            

              
Example .gitignore

              
node_modules/
dist/
.env
.DS_Store
